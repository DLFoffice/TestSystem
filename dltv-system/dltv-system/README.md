# ระบบกรอกข้อมูลประเมินคุณภาพโรงเรียนปลายทาง DLTV

ระบบกรอกแบบฟอร์มประเมิน DLTV (DLTV-01 ถึง DLTV-07) แบบ data-driven
นิยามแต่ละแบบฟอร์มไว้เป็นไฟล์ JSON แยก แก้ไขเนื้อหาได้โดยไม่ต้องแตะโค้ด
รองรับ dashboard สรุปผล, การแก้ไข/ลบข้อมูล, ส่งออก PDF และบันทึกขึ้น Firebase Firestore

## มีอะไรใหม่ในเวอร์ชันนี้

- **DLTV-07 — แบบสรุปผลรวมของครูปลายทาง** ดึงผลการประเมิน DLTV-02 ของโรงเรียนที่เลือกมาโดยอัตโนมัติ
  (สูงสุด 3 ครู) คำนวณค่าเฉลี่ยรายตัวชี้วัดและคะแนนรายองค์ประกอบให้สด ๆ แก้ไข บันทึก และส่งออก PDF ได้
- **แดชบอร์ดสรุปผล** ภาพรวมคะแนนรายโรงเรียนตามแบบฟอร์มต่าง ๆ พร้อมแถบความก้าวหน้าและสีตามระดับคุณภาพ
- **แก้ไข / ลบข้อมูล** ทุกระเบียนที่บันทึกไว้ แก้ไขได้ (เปิดฟอร์มเดิมพร้อมข้อมูลที่กรอกไว้) และลบได้
- **ส่งออก PDF** ทุกแบบฟอร์ม รวมถึงแดชบอร์ดและ DLTV-07 (ใช้การพิมพ์ของเบราว์เซอร์ → เลือก "บันทึกเป็น PDF")
- **บันทึกถาวรในเครื่อง (localStorage)** ในโหมดออฟไลน์ ข้อมูลไม่หายเมื่อรีเฟรช และนำเข้า/ส่งออกเป็น JSON ได้

## โครงสร้างไฟล์

```
dltv-system/
├── index.html              หน้าหลัก
├── css/
│   └── styles.css          สไตล์ทั้งหมด
├── js/
│   ├── firebase-config.js  ★ ใส่ค่าโปรเจกต์ Firebase ที่นี่
│   ├── firebase-service.js  บันทึก/ดึง/แก้ไข/ลบข้อมูล (โหมดออฟไลน์เก็บใน localStorage)
│   ├── form-renderer.js     สร้างฟอร์มจาก JSON + คิดคะแนน + เติมค่ากลับเพื่อแก้ไข
│   └── app.js               ตัวควบคุมหลัก (ฟอร์ม/แดชบอร์ด/สรุปผล/PDF)
└── forms/                  ★ เนื้อหาแบบฟอร์ม (แก้ไขได้ง่าย)
    ├── index.json           ทะเบียนรวมแบบฟอร์ม
    ├── common.json          ข้อมูลพื้นฐานโรงเรียน
    ├── dltv01.json          แบบสำรวจการบริหารจัดการ โดย ผอ. (ไม่มีคะแนน)
    ├── dltv02.json          ประเมินคุณภาพโรงเรียนปลายทาง โดย ผอ. (99 คะแนน · 3 องค์ประกอบ)
    ├── dltv03.json          สำรวจการจัดการเรียนรู้ ครูปลายทาง (99 คะแนน + ปลายเปิด)
    ├── dltv05.json          ประเมินครูห้องเรียนต้นทาง (75 คะแนน)
    ├── dltv06.json          ครูต้นทางประเมินตนเอง (75 คะแนน)
    └── dltv07.json          แบบสอบถามนักเรียนปลายทาง (30 คะแนน · 10 ข้อ)
```

ไฟล์ `DLTV-ระบบกรอกข้อมูล.html` เป็นเวอร์ชันรวมไฟล์เดียว (single-file) ที่รวม CSS, JSON และ JS
ทั้งหมดไว้ในไฟล์เดียว เปิดได้ทันทีโดยดับเบิลคลิก ไม่ต้องใช้เว็บเซิร์ฟเวอร์

## การใช้งาน

### วิธีที่ 1 — ไฟล์เดียวจบ (ง่ายสุด)

ดับเบิลคลิก `DLTV-ระบบกรอกข้อมูล.html` เปิดในเบราว์เซอร์ได้เลย ทำงานออฟไลน์
ข้อมูลเก็บใน localStorage ของเบราว์เซอร์เครื่องนั้น

### วิธีที่ 2 — แบบแยกไฟล์ (สำหรับแก้ไข/พัฒนาต่อ)

ค่าเริ่มต้น `OFFLINE_MODE = true` ระบบทำงานได้เลยโดยไม่ต้องต่อ Firebase
ต้องเปิดผ่านเว็บเซิร์ฟเวอร์ (เพราะใช้ ES modules + fetch) ห้ามเปิดไฟล์ตรง ๆ:

```bash
cd dltv-system
python3 -m http.server 8000
# เปิด http://localhost:8000
```

> เมื่อแก้ไฟล์ในโฟลเดอร์ `css/ js/ forms/` แล้วต้องการสร้างไฟล์รวมใหม่
> ให้รัน `python3 build_single.py` จะได้ `DLTV-ระบบกรอกข้อมูล.html` ที่อัปเดตตาม

## ส่งออก PDF

ทุกหน้า (ฟอร์ม / แดชบอร์ด / DLTV-07) มีปุ่ม **"ส่งออก PDF"**
เมื่อกดจะเปิดหน้าจัดรูปแบบสำหรับพิมพ์พร้อมหัวกระดาษและช่องลงนาม จากนั้นเลือกปลายทางการพิมพ์เป็น
**"บันทึกเป็น PDF" (Save as PDF)** วิธีนี้รองรับฟอนต์ภาษาไทยได้ครบถ้วนโดยไม่ต้องใช้ไลบรารีภายนอกหรืออินเทอร์เน็ต

## แดชบอร์ดและการแก้ไขข้อมูล

- **แดชบอร์ดสรุปผล** อยู่ในเมนูกลุ่ม "สรุปผล & รายงาน" แสดงการ์ดรายโรงเรียน คะแนนรวม
  เปอร์เซ็นต์ และคะแนนแยกตามองค์ประกอบ
- **ข้อมูลที่บันทึก** แสดงทุกระเบียน กรองตามแบบฟอร์มได้ แต่ละแถวมีปุ่ม ส่งออก PDF / แก้ไข / ลบ
  และมีปุ่มนำเข้าไฟล์ JSON เพื่อกู้คืนหรือรวมข้อมูลจากเครื่องอื่น

## เชื่อมต่อ Firebase (ใช้งานจริง)

1. สร้างโปรเจกต์ที่ https://console.firebase.google.com แล้วเปิดใช้ **Firestore Database**
2. เพิ่ม Web App แล้วคัดลอกค่า config มาใส่ใน `js/firebase-config.js`
3. ตั้ง `OFFLINE_MODE = false`

ตัวอย่างกฎความปลอดภัย Firestore (ปรับตามนโยบายจริง — เวอร์ชันนี้รองรับแก้ไข/ลบโดยผู้ดูแล):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /dltv_submissions/{doc} {
      allow create: if true;                 // ให้กรอกส่งได้
      allow read: if request.auth != null;   // อ่านได้เฉพาะผู้ล็อกอิน
      allow update, delete: if request.auth != null; // แก้ไข/ลบโดยผู้ดูแล
    }
  }
}
```

## การแก้ไข/เพิ่มแบบฟอร์ม

แก้เนื้อหาข้อคำถามได้ที่ไฟล์ใน `forms/` โดยตรง ชนิดฟิลด์ที่รองรับ:
`text`, `number`, `textarea`, `select`, `radio`, `checkbox`,
`rating` (ให้คะแนน 3/2/1), `table` (ตารางกรอกข้อมูล), `level`, `heading`, `note`

- ทำให้ section คิดคะแนน: ใส่ `"scored": true` และ `"maxScore"`
- กำหนดระดับคุณภาพ: ใส่ `"qualityBands"` (เช่น ดี/พอใช้/กำลังพัฒนา ตาม DLTV-00)
- เพิ่มฟอร์มใหม่: สร้างไฟล์ใน `forms/` แล้วเพิ่มรายการใน `forms/index.json`
- ฟอร์มสรุปผล (เช่น DLTV-07): ใช้ `"kind": "summary"` และระบุ `"source"` เป็นฟอร์มต้นทาง
